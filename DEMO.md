# SentinelPhish Demo Protocol

**Professional 60-Second Sales Walkthrough Guide**

---

## Overview

This document provides a structured protocol for creating a high-quality, 60-second sales demonstration of SentinelPhish. The demo is designed to showcase the tool's effectiveness in detecting phishing threats without revealing source code or internal implementation details.

---

## Pre-Demo Checklist

### Environment Setup

- [ ] Clean browser window (incognito mode recommended)
- [ ] All API keys configured in `.env.local`
- [ ] Development server running: `npm run dev`
- [ ] Application accessible at `http://localhost:3000`

### Test URLs Prepared

**Safe URL (Control)**
- Recommended: `https://google.com` or `https://microsoft.com`
- Alternative: Any well-known legitimate domain

**Phishing URL (Test)**
- Use a known phishing test site from PhishTank
- OR create a controlled test environment
- OR use a URL with obvious phishing indicators (for demonstration only)

### Recording Setup

- [ ] Screen recording software installed (OBS, Loom, etc.)
- [ ] Resolution set to 1080p or 4K
- [ ] Audio configured (microphone or voiceover)
- [ ] Browser window maximized
- [ ] All other tabs closed

---

## 60-Second Demo Script

### Segment 1: Introduction (0:00-0:10)

**Visuals:**
- Show full application interface
- Clean, professional view of the scanning page

**Script (Voiceover/On-Screen Text):**
> "Welcome to SentinelPhish, the AI-powered phishing detection tool that protects your organization in seconds."

**Actions:**
- Smooth pan across the interface
- Highlight the URL input field
- No clicks yet

### Segment 2: Safe URL Test (0:10-0:25)

**Visuals:**
- Type safe URL (google.com)
- Click "Scan URL" button
- Show loading animation
- Display results

**Script:**
> "Let's test with a known safe domain. Enter google.com, click scan, and see the result: Risk score of 15 - SAFE."

**Actions:**
- Type URL smoothly
- Click button
- Wait for results
- Highlight green status indicator
- Point to "No red flags" message

### Segment 3: Phishing URL Test (0:25-0:45)

**Visuals:**
- Type phishing test URL
- Click "Scan URL" button
- Show loading animation
- Display results with red indicators

**Script:**
> "Now let's test a suspicious site. Enter [test-phishing-url], click scan. SentinelPhish immediately detects multiple threats: Risk score of 85 - DANGEROUS."

**Actions:**
- Type URL smoothly
- Click button
- Wait for results
- Highlight red status indicator
- Scroll through risk flags
- Point to AI analysis section

### Segment 4: AI Analysis Highlight (0:45-0:52)

**Visuals:**
- Focus on AI analysis section
- Show screenshot of the phishing site
- Display Gemini AI verdict

**Script:**
> "Our AI-powered visual analysis detected a credential harvester on this untrusted domain."

**Actions:**
- Hover over screenshot
- Highlight AI verdict text
- Show actionable advice section

### Segment 5: Conclusion (0:52-1:00)

**Visuals:**
- Show download report button
- Return to main interface
- Fade to logo/contact info

**Script:**
> "Get actionable intelligence in seconds. Protect your organization today with SentinelPhish."

**Actions:**
- Hover over "Download Report" button
- Fade transition
- Display contact information

## Recording Best Practices

### Visual Quality

- **Resolution**: Minimum 1080p, preferred 4K
- **Frame Rate**: 30fps or 60fps
- **Bitrate**: High quality (5-10 Mbps for 1080p)
- **Color**: Accurate color reproduction

### Audio Quality

- **Voiceover**: Clear, professional tone
- **Background Music**: Subtle, non-distracting (optional)
- **Sound Effects**: Minimal, only for emphasis
- **Volume**: Consistent levels

### Mouse Movement

- **Speed**: Smooth, deliberate movements
- **Path**: Direct, minimal wandering
- **Hover**: Brief pauses on key elements
- **Clicks**: Precise timing

### Typography

- **On-Screen Text**: Clear, readable fonts
- **Contrast**: High contrast for readability
- **Size**: Large enough for small screens
- **Placement**: Strategic, not cluttered

## Troubleshooting

### Common Issues

**Issue**: Scan takes too long
- **Solution**: Use cached results or pre-record segments

**Issue**: AI analysis fails
- **Solution**: Have backup screenshots prepared
- **Alternative**: Skip AI analysis segment

**Issue**: URL doesn't load
- **Solution**: Use alternative test URLs
- **Preparation**: Have 3-5 backup URLs ready

**Issue**: Recording quality poor
- **Solution**: Test recording setup beforehand
- **Check**: Lighting, audio, screen resolution
